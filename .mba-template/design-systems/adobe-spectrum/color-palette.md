# Adobe Spectrum - Color Palette

## Overview
Adobe Spectrum uses a semantic color system with specific roles for UI elements.

## Color Roles

### Semantic Colors
- **Gray**: Neutral backgrounds, borders, disabled states
- **Blue**: Primary actions, links, focus states
- **Red**: Errors, destructive actions, warnings
- **Orange**: Warnings, notices
- **Green**: Success states, positive feedback
- **Indigo**: Selected states, highlights

## Color Scale
Each color has 14 shades (100-1400) providing granular control:
- 100-300: Very light (backgrounds, subtle states)
- 400-600: Medium (borders, secondary elements)
- 700-900: Standard (primary text, icons)
- 1000-1400: Dark (emphasis, high contrast)

## Usage Principles
1. **Semantic First**: Use colors for their meaning, not decoration
2. **Contrast**: Maintain WCAG AA minimum (4.5:1 for text)
3. **Consistency**: Same color role = same meaning across app
4. **Accessibility**: Test with color blindness simulators

## Background Colors
- **Gray-50**: Lightest background
- **Gray-75**: Subtle background variation
- **Gray-100**: Default background
- **Gray-200**: Elevated surface (cards, modals)

## Text Colors
- **Gray-900**: Primary text (high emphasis)
- **Gray-700**: Secondary text (medium emphasis)
- **Gray-600**: Tertiary text (low emphasis)
- **Gray-400**: Disabled text

## Border Colors
- **Gray-300**: Default borders
- **Gray-400**: Hover borders
- **Gray-500**: Active/focus borders

## Action Colors
- **Blue-600**: Primary button background
- **Blue-700**: Primary button hover
- **Red-600**: Destructive action
- **Green-600**: Success confirmation
